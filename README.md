
# CENA - Plataforma de Self-Tape para Atores

Bem-vindo à CENA, uma plataforma SaaS inovadora desenhada para atores que buscam aprimorar suas habilidades de self-tape, receber feedback construtivo de tutores qualificados e gerenciar seu progresso artístico e de carreira. A plataforma também oferece ferramentas robustas para administradores gerenciarem conteúdo, usuários e funcionalidades, e para tutores acompanharem seus alunos e fornecerem avaliações detalhadas.

## Visão Geral

A CENA visa ser o ambiente digital completo para o desenvolvimento do ator moderno. Com um foco em prática contínua, feedback de qualidade e acesso a recursos educacionais, a plataforma oferece uma experiência integrada para todos os seus usuários.

## Funcionalidades Principais

A plataforma é dividida em diferentes painéis e funcionalidades, dependendo do papel do usuário:

### Para Atores (Ator/Atriz)

*   **Em Cena (Painel Principal):** Um hub central para acesso rápido às principais funcionalidades, dicas e novidades.
*   **Tema do Mês:** Acesso a um tema mensal proposto pela plataforma, incluindo materiais de estudo (PDFs, vídeos de referência) para inspirar a criação de self-tapes.
*   **Envio de Self-Tapes:** Ferramenta para enviar vídeos de self-tape (links do YouTube) baseados no tema do mês ou em textos livres (conforme o plano).
*   **Meus Feedbacks:** Visualização detalhada dos feedbacks recebidos dos tutores, incluindo vídeo de análise e transcrição/comentários.
*   **Sessões 1:1:**
    *   Listagem de categorias de sessões individuais disponíveis (Ex: Análise de Texto, Técnica Vocal, Preparação para Testes).
    *   Visualização de detalhes de cada categoria e simulação de solicitação de agendamento.
    *   Acompanhamento de sessões agendadas com tutores (simulado).
*   **Percursos (Cursos e Workshops):**
    *   Listagem de cursos e workshops disponíveis (online ao vivo, presenciais, gravados).
    *   Visualização de detalhes de cada percurso, incluindo descrição, instrutor, preço, e materiais.
    *   Simulação de inscrição em percursos.
*   **Ao Vivo:**
    *   Visualização de eventos públicos da plataforma (workshops, masterclasses) com calendário.
    *   Acesso a links para participação (simulado).
*   **Mensagens:** Central de notificações da plataforma (novos temas, feedbacks prontos, eventos, etc.) e um canal de contato simulado com o suporte.
*   **Meu Perfil:**
    *   Gerenciamento completo do perfil, incluindo dados pessoais, foto, nível de atuação, escolaridade, experiências formativas e profissionais, objetivos e interesses técnicos.
    *   Visualização e simulação de alteração do plano de assinatura e ciclo de pagamento.
    *   Visualização da "Trajetória CENA" com estatísticas de progresso (self-tapes enviados, feedbacks recebidos, etc.).

### Para Tutores

*   **Painel do Tutor:** Visão geral com estatísticas chave (revisões pendentes, atores designados, feedbacks enviados).
*   **Envios para Feedback:**
    *   Visualização de self-tapes de atores pendentes de feedback, tanto os designados diretamente ao tutor quanto os disponíveis no "pool geral".
    *   Possibilidade de assumir envios do pool.
    *   Detalhes de cada envio, incluindo informações do ator, do tema, e prazo para feedback.
*   **Dar Feedback:** Interface para fornecer feedback para um envio específico, incluindo campo para link de vídeo de feedback (YouTube) e transcrição/comentários.
*   **Feedbacks Concluídos:** Histórico de todos os feedbacks já fornecidos, agrupados por tema.
*   **Gerenciar Sessões Ao Vivo (Sessões 1:1):**
    *   Visualização e simulação de agendamento de sessões individuais com atores.
    *   Atualização do status das sessões (Concluída, Cancelada).
    *   Acesso também aos eventos públicos da plataforma.
*   **Mensagens:** Acesso às notificações da plataforma e ao canal de suporte.
*   **Meu Perfil:** Edição das informações de perfil do tutor, similar ao formulário de candidatura.

### Para Administradores (Admin)

*   **Painel do Administrador:** Visão geral das principais áreas de gerenciamento.
*   **Gerenciar Temas:** Criação, edição (título, descrição, mês/ano, imagem de cabeçalho, vídeo de referência, PDFs), ativação/desativação de temas mensais.
    *   Funcionalidade de geração de descrição de temas com IA (Google Gemini).
*   **Gerenciar Percursos:** Criação, edição (título, descrição, instrutor, tipo, preço, imagem, etc.), publicação/ocultação de cursos e workshops.
    *   Controle da visibilidade da página inteira de "Percursos" para os atores.
*   **Gerenciar Sessões 1:1:** Criação, edição e gerenciamento das categorias de sessões individuais oferecidas na plataforma.
*   **Gerenciar Landing Page:** Edição de todos os textos da página inicial pública da plataforma.
*   **Gerenciar Usuários:**
    *   Listagem e visualização de todos os usuários (Atores, Convidados, Admins).
    *   Edição de informações básicas do usuário (papel principal, plano do ator, status da conta do ator).
    *   Visualização do dossiê de atores.
    *   Aprovação/suspensão de acesso para Convidados.
*   **Gerenciar Tutores:**
    *   Listagem de tutores por status (Ativos, Em Observação, Candidaturas Pendentes, Inativos/Rejeitados).
    *   Visualização do dossiê e detalhes da candidatura de cada tutor.
    *   Aprovação, rejeição, movimentação para observação e (des)ativação do acesso de tutores.
*   **Atribuição de Envios:** Painel para visualizar self-tapes pendentes e designá-los manualmente para tutores disponíveis, ou retornar envios designados para o pool geral.
*   **Gerenciar Menus Laterais (Sidebars):** Ferramenta para personalizar os menus laterais para cada papel (Ator, Tutor, Convidado):
    *   Ocultar/mostrar itens.
    *   Ativar/desativar (tornar não clicável) itens.
    *   Reordenar itens via drag-and-drop.
    *   Restaurar a configuração padrão para um papel.
*   **Enviar Notificações (Painel de Mensagens):** Ferramenta para enviar notificações em massa para diferentes segmentos de usuários (Todos, Atores, Tutores Aprovados, Convidados Aprovados) ou segmentar atores por status da assinatura, plano, nível e técnicas de interesse.
*   **Ao Vivo:** Gerenciamento de eventos públicos da plataforma (criação, edição, exclusão, definição de público alvo).
*   **Mensagens:** Acesso às suas próprias notificações.
*   **Meu Perfil (Admin):** Visualização do perfil do administrador com estatísticas da plataforma.

### Para Convidados (Guest)

*   Acesso limitado para explorar o Tema do Mês e os Percursos publicados.
*   Acesso à página "Ao Vivo" e "Mensagens".
*   Visualização do próprio perfil.
*   O acesso completo depende da aprovação do administrador.

### Para Visitantes (Não Logados)

*   **Landing Page:** Página de apresentação da plataforma CENA.
*   **Página de Planos:** Detalhamento dos planos de assinatura e add-ons.
*   Funcionalidades de Registro e Login.

## Como Executar o Projeto Localmente

Este projeto foi desenvolvido para ser executado diretamente no navegador sem a necessidade de um passo de build complexo ou um servidor Node.js para desenvolvimento básico.

### Pré-requisitos

*   Um navegador web moderno que suporte ES Modules e Import Maps (Chrome, Edge, Firefox, Safari recentes).
*   Acesso à internet para carregar as dependências do React e outras bibliotecas via CDN (esm.sh).

### Passos para Execução

1.  **Obter os Arquivos:**
    *   Se você clonou de um repositório Git: `git clone <url_do_repositorio>`
    *   Se você recebeu os arquivos diretamente: Descompacte-os em uma pasta no seu computador.

2.  **Abrir no Navegador:**
    *   Navegue até a pasta do projeto no seu explorador de arquivos.
    *   Abra o arquivo `index.html` diretamente no seu navegador.

3.  **(Recomendado para Desenvolvimento) Usar um Live Server:**
    *   Para uma melhor experiência de desenvolvimento com recarregamento automático (hot-reload) ao salvar alterações nos arquivos, recomenda-se usar uma extensão de live server no seu editor de código (ex: "Live Server" para VS Code).
    *   Com a extensão instalada, clique com o botão direito no arquivo `index.html` e selecione a opção "Open with Live Server" (ou similar). Isso abrirá o projeto em um servidor local (ex: `http://127.0.0.1:5500/index.html`).

## Configuração da API Key do Google Gemini

A funcionalidade de "Gerar com IA" para a descrição de temas na página de criação/edição de temas utiliza a API do Google Gemini.

*   **Requisito:** Para que esta funcionalidade opere, uma API Key válida do [Google AI Studio](https://aistudio.google.com/app/apikey) é necessária.

*   **Como Configurar (Importante!):**
    *   Conforme as diretrizes de desenvolvimento, a API Key **DEVE** ser obtida através da variável de ambiente `process.env.API_KEY`.
    *   **Desafio:** Este projeto, ao ser executado diretamente no navegador (`index.html`), não tem acesso direto a variáveis de ambiente do sistema da mesma forma que um projeto Node.js ou um projeto com build step teria.
    *   **Solução para Ambientes como Replit:**
        1.  No Replit (ou ambiente similar que suporte "secrets"), adicione sua API Key do Google Gemini como um "Secret". O nome da variável secreta deve ser `API_KEY`.
        2.  Para que o código frontend (`ThemeFormPage.tsx`, que tenta acessar `process.env.API_KEY`) possa usar esta chave, você precisaria de um mecanismo para que o Replit exponha este "secret" ao JavaScript do lado do cliente.
            *   O código em `ThemeFormPage.tsx` já possui uma verificação: `if (typeof process !== 'undefined' && process.env && process.env.API_KEY)`. Se `process.env.API_KEY` não estiver disponível, a funcionalidade de IA é desabilitada com um aviso, e a aplicação não quebra.
            *   Você precisará investigar como o Replit (ou seu ambiente de hospedagem) pode injetar "secrets" de forma que sejam acessíveis como `process.env.API_KEY` no escopo do navegador, ou adaptar o código para buscar essa chave de uma forma segura que o ambiente de hospedagem forneça (ex: através de um objeto global injetado pelo Replit ou um endpoint backend seguro, embora a segunda opção adicione complexidade não presente no projeto atual).
            *   Uma forma **simples (mas menos segura para produção real e dependente do ambiente)** seria injetar a chave no `index.html` antes do script principal, se o ambiente permitir:
                ```html
                <!-- index.html (Exemplo Conceitual - NÃO FAÇA ISSO COM CHAVES REAIS EM PRODUÇÃO PÚBLICA) -->
                <script>
                  // Assume que REPLIT_API_KEY é uma variável que o Replit tornaria disponível
                  // window.process = window.process || {};
                  // window.process.env = window.process.env || {};
                  // window.process.env.API_KEY = "SUA_API_KEY_AQUI_SE_O_REPLIT_NAO_INJETAR_AUTOMATICAMENTE";
                </script>
                <script type="module" src="./index.js"></script> <!-- ou index.tsx se for o caso -->
                ```
            *   **A recomendação é sempre priorizar os mecanismos de "secrets" do ambiente de hospedagem e adaptar o acesso à chave no frontend da forma mais segura possível que o ambiente permitir.**

*   **Sem a Chave:** Se a API Key não estiver configurada ou acessível, a funcionalidade de geração de descrição por IA na página de temas será desabilitada, mas o restante da plataforma funcionará normalmente.

## Executando no Replit

Este projeto foi construído como uma aplicação frontend estática e pode ser executado no Replit com algumas configurações e considerações:

1.  **Importar para o Replit:**
    *   Você pode importar o projeto diretamente de um repositório GitHub para o Replit.

2.  **Configuração de Execução do Repl:**
    *   **Tipo de Repl:** Ao importar, o Replit pode tentar configurar um ambiente Node.js por padrão. Para este projeto, o ideal é que o Replit o trate como um site estático. Se possível, durante ou após a importação, configure o tipo de Repl para **"HTML, CSS, JS"** ou **"Static Site"**. Isso geralmente configura o ambiente de execução corretamente para servir o `index.html`.
    *   **Arquivo `.replit`:** Se o Replit criar ou você tiver um arquivo `.replit` na raiz, certifique-se de que ele esteja configurado para servir o `index.html`.
        *   Remova ou comente qualquer linha como `run = "npm run dev"` ou `run = "npm start"`, pois não usamos `npm` para executar este projeto.
        *   O `entrypoint` deve ser `index.html`. Exemplo de um `.replit` minimalista para um site estático:
            ```toml
            language = "html"
            entrypoint = "index.html"
            # Quaisquer outros comandos 'run' desnecessários devem ser removidos.
            ```
    *   **Comando `Run`:** Com a configuração correta do tipo de Repl ou do `.replit`, o botão "Run" no Replit deve iniciar um servidor web simples que serve o `index.html`.

3.  **Configurar API Key do Google Gemini (para funcionalidade de IA):**
    *   A funcionalidade de "Gerar com IA" para a descrição de temas na página de administração de temas utiliza a API do Google Gemini.
    *   **Secrets do Replit:**
        1.  No seu Repl, vá para a seção "Secrets" (geralmente um ícone de cadeado na barra lateral esquerda).
        2.  Adicione um novo secret com o nome (Key) `API_KEY`.
        3.  No campo "Value", cole sua chave da API do Google Gemini.
    *   **Acesso no Código:** O arquivo `index.html` deste projeto inclui um script no `<head>` que tenta ler o `API_KEY` dos "Secrets" do Replit e torná-lo disponível para o JavaScript da aplicação como `window.process.env.API_KEY`.
        ```html
        <!-- Dentro do <head> no index.html -->
        <script>
          fetch('/__replenv.json') // Endpoint padrão do Replit para secrets
            .then(response => response.ok ? response.json() : {})
            .then(env => {
              window.process = window.process || {};
              window.process.env = window.process.env || {};
              if (env.API_KEY) {
                window.process.env.API_KEY = env.API_KEY;
                console.log('API_KEY for Gemini loaded from Replit Secrets.');
              } else {
                console.warn('API_KEY for Gemini not found in Replit Secrets. AI features might be disabled.');
              }
            })
            .catch(error => {
              console.warn('Could not fetch Replit environment variables (this is normal if not running on Replit or if __replenv.json is not available). AI features might be disabled.', error);
              window.process = window.process || {};
              window.process.env = window.process.env || {};
            });
        </script>
        ```
        O código em `ThemeFormPage.tsx` já está preparado para usar `process.env.API_KEY` e desabilitar a funcionalidade de IA se a chave não estiver disponível.

4.  **Dependências:**
    *   Este projeto carrega suas dependências principais (React, React Router, etc.) via CDN através de um `importmap` no `index.html`. Você **não precisa** executar `npm install` no Replit para essas dependências.

5.  **Acessar a Aplicação:**
    *   Após o Replit iniciar e servir os arquivos, sua aplicação estará disponível na URL fornecida pelo Replit (geralmente exibida em uma aba "Webview" ou "Output" dentro do ambiente Replit, ou como um link `*.replit.dev`).

## Considerações Adicionais

*   **Mock Backend:** Toda a persistência de dados é feita usando `localStorage`. Isso significa que os dados são armazenados localmente no navegador do usuário. Para uma aplicação de produção, um backend real com um banco de dados seria necessário.
*   **Autenticação Simulada:** O sistema de login e registro é simulado e não envolve comunicação com um servidor de autenticação real.
*   **Responsividade:** A plataforma foi desenhada com Tailwind CSS, visando uma boa experiência em desktops, tablets e dispositivos móveis.
*   **Acessibilidade (ARIA):** Tentou-se seguir boas práticas de acessibilidade, mas revisões e testes adicionais são sempre bem-vindos.
